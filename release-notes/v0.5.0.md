# better-git-aliases v0.5.0

This release starts the next major direction for the project:

- host-aware remote actions
- safer provider detection
- no change to the Git-native core workflow

## Highlights

- `git remote-open`
  - opens the current remote's parsed web URL
- `git remote-compare [base] [compare]`
  - prints compare URLs for supported providers and falls back safely when the provider is not yet supported
- formal `v0.5.0` roadmap
  - clarifies that provider-aware features stay optional and should not take over the main workflow
