# better-git-aliases v0.2.0

First feature-focused release after the initial launch.

This version makes the toolkit more usable as a daily driver for multi-feature work, tmux-based workflows, and safer lazygit setup management.

## Highlights

- New tmux cockpit layouts:
  - `solo`
  - `review`
  - `ai-pair`
  - `multi-feature`
- Safer lazygit config management:
  - `git lazygit-config diff`
  - `git lazygit-config backup`
  - `git lazygit-config merge-hint`
- Worktree lifecycle helpers:
  - `git feature finish`
  - `git wt stale`
  - `git wt clean`
- Stronger `git doctor` diagnostics with clearer remediation guidance

## Why this release matters

`v0.2.0` moves the project from "a useful alias bundle" toward "a practical daily Git workflow toolkit".

The main gains are:

- easier tmux session reuse
- better multi-feature cleanup
- safer lazygit adoption for users with existing config
- clearer setup diagnostics for normal users

## Quick examples

```bash
git cockpit list-layouts
git cockpit --layout review
git lazygit-config diff
git wt stale
git wt clean --apply
git feature finish my-feature --base main
```
