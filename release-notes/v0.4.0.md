# better-git-aliases v0.4.0

This release expands the toolkit from a single-repo daily workflow into a practical multi-repo workspace workflow.

The main theme is workspace-aware Git work:

- see several repos at once
- sync several repos safely
- jump into the right repo action quickly
- share a few repo defaults with a team without giving up personal overrides

## Highlights

- `git wsdash`
  - shows branch, dirty state, upstream state, and worktree count across repos
- `git wssync`
  - fetches all repos in a workspace and rebases branches that already have an upstream
- `git wsmenu`
  - lets you pick a repo and then choose dashboard, sync, lazygit, cockpit, or AI actions
- `git config-bga`
  - manages personal defaults such as `base_branch`, `remote`, `cockpit_layout`, and `workspace_root`
- repo-level `.bga.env`
  - gives teams a simple way to ship shared defaults in the repo itself

## Why this release matters

`v0.4.0` turns better-git-aliases into something more useful for engineers who juggle multiple repos in one day.

It also makes the toolkit easier to standardize inside one team:

- the repo can define sane defaults
- individual developers can still override them locally
- the main workflow stays Git-native and host-agnostic

## Quick examples

```bash
git config-bga set workspace_root ~/src
git wsdash
git wsmenu
git wssync
```

To add repo-level conventions:

```bash
cp .bga.env.example .bga.env
```
