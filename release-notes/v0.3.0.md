# better-git-aliases v0.3.0

This release turns the project into a more complete daily workflow toolkit.

The main theme is daily workflow automation:

- start the day faster
- resume feature work more easily
- submit features with less friction
- clean up after merge with a safer flow

## Highlights

- `git start-day`
  - runs doctor, sync, and dashboard in sequence
- `git feature resume`
  - resume a feature worktree directly
- `git feature submit`
  - now platform-agnostic
  - publishes the branch and tells you to create the PR/MR in your web UI
- `git feature cleanup`
  - previews and applies post-merge cleanup
- stronger task-oriented menu
- stronger dashboard attention signals

## Why this release matters

`v0.3.0` is where the project becomes less about isolated helpers and more about full workflow loops:

- start work
- resume work
- review and submit
- clean up after merge

That is a more durable improvement than simply adding more aliases.

## Quick examples

```bash
git start-day
git feature resume login-page --cockpit
git feature submit
git feature cleanup --apply
```
