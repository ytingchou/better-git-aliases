# better-git-aliases v0.4.2

This patch release improves two practical edges of the toolkit:

- preparing the next release
- understanding the current Git remote across different hosting providers

## Highlights

- `git release-init <version>`
  - scaffolds a new release note file from commits since the latest version tag
- `git remote-info [remote]`
  - parses the current remote into provider, host, path, web URL, and raw Git URL
- Azure DevOps remote detection
  - supports both SSH and HTTPS URL styles without changing the main Git-native workflow
