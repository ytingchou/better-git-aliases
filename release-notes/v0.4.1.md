# better-git-aliases v0.4.1

This patch release tightens workflow safety and makes the project easier to try and release.

## Highlights

- `protected_branches`
  - lets teams protect long-lived local branches during cleanup
- `./scripts/demo-workspace.sh`
  - creates a throwaway multi-repo playground with `.bga.env`
- `git release-check`
  - verifies version consistency across `VERSION`, `README`, `CHANGELOG`, and release notes

## Why this release matters

`v0.4.1` is about reducing friction around the workflow that `v0.4.0` introduced:

- safer cleanup defaults
- easier hands-on evaluation
- less manual release bookkeeping

## Quick examples

```bash
git config-bga set protected_branches main,master,develop,release
./scripts/demo-workspace.sh
git release-check
```
