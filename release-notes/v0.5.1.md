# better-git-aliases v0.5.1

This patch release improves the provider-aware remote workflow that started in `v0.5.0`.

## Highlights

- `git remote-status`
  - shows the current branch, base branch, provider, upstream state, and compare readiness
- remote actions in `git menu`
  - remote status, open repo, and open compare are now discoverable from the main menu
- formal `v0.6.0` roadmap
  - clarifies the next direction for provider-aware review context without forcing host-specific workflows
